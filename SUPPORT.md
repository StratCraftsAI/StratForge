# Support

## Where to Ask for Help

- **Bug reports**: Open a GitHub issue with a minimal reproduction
- **Security issues**: Follow [SECURITY.md](SECURITY.md) and report privately by email
- **Feature ideas**: Open an issue before starting implementation
- **Usage questions**: Prefer GitHub issues until Discussions are enabled

## What to Include

For build or runtime issues, include:

- Operating system
- Compiler and version
- CMake version
- Build type (`Release` or `Debug`)
- Minimal code sample or dataset that reproduces the problem
- Exact compiler or test output

## Response Expectations

- Best-effort support for the latest public release
- Older releases may receive limited guidance
- Performance questions should include benchmark context and hardware details
