# Roadmap

## Near Term

- Stabilize public packaging and release flow for downstream CMake consumers
- Expand non-indicator test coverage called out in `docs/tickets/`
- Tighten benchmark regression tracking and baseline publication
- Improve public examples for optimizer, replay, and multi-data workflows

## Mid Term

- Clarify public API stability boundaries across headers
- Add broader observer and analyzer validation coverage
- Document extension patterns for custom indicators, analyzers, and data feeds

## Longer Term

- Improve ecosystem integration for larger quant stacks
- Add stronger upgrade guidance between public releases
- Expand contributor automation and repository health checks
